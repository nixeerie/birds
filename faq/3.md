*Otázky*
--------

1. Jaký typ SQL dotazu se použije ve vaší aplikaci pro založení nového záznamu do datasetu?
2. Napište dekorátor, který používá funkce implementující obsluhu mazání záznamů z databáze.
3. Popište, jakým způsobem se nahrají do databáze počáteční data. Neodpovídejte však triviálním tvrzením, např. spustí se tento skript. Odpovězte, kde nebo jak jsou zdrojová data uložena, a co dělá skript, který je importuje do databáze.
4. Jak jste vyřešili přístup uživatelů do aplikace?
5. Jakým způsobem ukládáte hesla?
6. Jak je řešeno, že danou operaci, např. úprava záznamu, může provést pouze přihlášený uživatel? Vložte ukázku kódu, který to řeší.

*Odpovědi*
----------
1. INSERT — například `INSERT INTO ptaci (...) VALUES (...)`.

2. Dekorátor `with_db` — otevře `sqlite3.connect('ptaci.db')`, předá `conn` funkci, po vykonání provede `commit()` nebo `rollback()` a zavře spojení.

3. Počáteční data jsou v `dataset_ptaci_final.csv` v kořeni repozitáře; `import_csv.py` smaže starou `ptaci.db`, vytvoří tabulku, načte CSV přes `csv.DictReader`, převede datové typy a vloží řádky parametrizovanými `INSERT`y, pak provede `commit()`.

4. Přístup uživatelů: v aktuální verzi není žádná autentizace (operace jsou veřejné). Doporučení: použít `Flask-Login` nebo session‑based přihlášení.

5. Ukládání hesel: projekt zatím neobsahuje uživatele; pokud přidáte, ukládejte pouze hash hesla (např. `werkzeug.security.generate_password_hash` nebo `bcrypt`).

6. Kontrola přihlášení — stručný příklad dekorátoru (`session`): `def login_required(f): ... if not session.get('user_id'): return redirect(url_for('login'))` a použít `@login_required` nad routou.
